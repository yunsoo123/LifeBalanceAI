# 실제 사용자·투자자 관점 (User–Investor Lens)

**목적**: Plan/Design 단계에서 "실제 사용자"와 "투자자/검토자" 관점을 적용해 보완점을 찾고, 로드맵·기능 우선순위에 반영한다.

**사용 시점**: PDCA plan 작성 시, 로드맵/계획 정리 시, 또는 "어플 보완점 더 찾아줘" 요청 시 이 문서의 체크리스트를 적용한다.

---

## 1. 실제 사용자 관점 (Real User)

### 1.1 첫인상·가치 이해 (30초~1분)

- [ ] **앱을 처음 켰을 때** "이게 뭐하는 앱이지?"가 30초 안에 해소되는가?
- [ ] **타겟(ADHD형, 목표 많고 계획 시간 적음)** 에게 "나를 위한 앱이다"가 한 줄 문구나 첫 화면에서 전달되는가?
- [ ] **온보딩/빈 상태** 에서 "다음에 뭘 하면 돼?"가 명확한가? (CTA 한 개가 확실한가?)

### 1.2 일상 사용·재방문 (Retention)

- [ ] **매일/매주 열게 만드는 이유** 가 있는가? (알림, 리뷰 탭의 "이번 주 돌아보기", 캘린더 동기화 등)
- [ ] **데이터가 쌓일수록** 더 유용해지는가? (과거 일정·노트·리뷰가 다음 계획에 반영되는가?)
- [ ] **이탈 포인트**: 어디서 "귀찮다"고 끌 가능성이 높은가? (예: 구조화 2단계, 일정 수동 입력만 가능한 구간)

### 1.3 결제 전환 (Monetization)

- [ ] **무료로 써보고** "이거 돈 내고 쓰고 싶다"가 나오는 지점이 있는가?
- [ ] **Pro 혜택** 이 "더 많이 쓰고 싶을 때" 자연스럽게 노출되는가? (한도 초과 시, 리뷰 고급 인사이트 등)
- [ ] **가격 인지**: 무료 vs Pro 차이가 설정/프로필에서 한눈에 보이는가?

### 1.4 신뢰·안정성

- [ ] **로딩/에러** 시 "뭔가 하고 있다 / 왜 실패했는지"가 사용자에게 전달되는가?
- [ ] **데이터 손실** 에 대한 불안이 없는가? (동기화 표시, 오프라인 시 큐 등)
- [ ] **개인정보·권한** 요청이 필요한 만큼만 있고, 이유가 설명되는가?

---

## 2. 투자자·검토자 관점 (Investor / Reviewer)

### 2.1 시장·차별화

- [ ] **한 줄 피치**: "Mendly는 [타겟]이 [문제]를 [어떻게] 해결하는 [앱/서비스]다"가 명확한가?
- [ ] **경쟁 대비**: 캘린더/할일/노트 앱과 뭘로 구분되는가? (AI 일정 생성, 인박스→캘린더, 주간 리뷰 등)
- [ ] **확장성**: 한국만? 글로벌? B2C만? 구독 외 수익화(광고 등) 전략이 일관된가?

### 2.2 성장·지표

- [ ] **핵심 지표** 가 정의되어 있는가? (DAU/WAU, 유지율, 유료 전환율, AI 호출당 만족도 등)
- [ ] **측정 가능한가?** (이벤트 로깅, 대시보드, A/B 가능 여부)
- [ ] **다음 6개월 목표** 가 "기능 N개"가 아니라 "지표 X 달성"으로 잡을 수 있는가?

### 2.3 리스크·품질

- [ ] **기술 부채**: 실기기 크래시, 콜 스택 오류 등이 문서화·우선순위되어 있는가?
- [ ] **보안·규정**: API 키, RLS, 개인정보 처리 방침이 출시 수준인가?
- [ ] **팀/리소스**: 현재 계획이 1인 개발/소규모에 현실적인가?

---

## 3. 적용 방법 (PDCA와 연동)

1. **로드맵/계획 작성 시**: 위 체크리스트를 한 번 돌려, "미충족" 항목을 **보완점**으로 정리한다.
2. **Plan 단계**: 새 feature plan 작성 시 "실제 사용자·투자자 관점" 섹션을 넣고, 위 1~2개 질문에 답하게 한다.
3. **보완점 → PDCA 후보**: 나온 보완점을 기존 3.1(성능)·3.2(신규)·3.3(디자인) 항목에 매핑하거나, 새 feature plan으로 추가한다.
