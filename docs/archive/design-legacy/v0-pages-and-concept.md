# v0.dev 페이지 목록 및 디자인 컨셉 (Mendly / LifeBalanceAI)

v0.dev로 화면을 만들 때 참고할 **페이지 개수**, **역할**, **컨셉**을 정리한 문서입니다.

---

## 전체 디자인 컨셉 (공통)

- **스타일**: 토스(Toss)·노션(Notion) 수준의 세련된 프로덕션 UI. 텍스트만 나열된 느낌 없이, 정보는 카드로 묶고 위계를 분명히.
- **배경**: 화면 전체 `bg-slate-50` (라이트), `bg-zinc-950` (다크).
- **카드**: `bg-white` / `dark:bg-zinc-900`, `rounded-2xl`, `p-5`, `shadow-sm`, `border border-gray-100` / `dark:border-zinc-800`. 연관된 정보는 반드시 카드로 그룹화.
- **정렬**: 라벨·값·설정 항목은 `flex-row items-center justify-between` 양끝 정렬.
- **타이포**  
  - 페이지 최상단 타이틀: `text-3xl font-extrabold text-gray-900` / `dark:text-gray-50`, `mb-6`  
  - 섹션 타이틀(카드 내): `text-lg font-bold text-gray-800` / `dark:text-gray-100`, `mb-4`  
  - 본문: `text-base text-gray-600` / `dark:text-gray-300`  
  - 보조/캡션: `text-[13px] text-gray-400` / `dark:text-gray-500`
- **입력·버튼**: 큰 입력창은 `rounded-2xl`, `p-5`, `min-h-[160px]` 수준. 버튼은 primary/outline/ghost 구분, `rounded-xl`·`rounded-2xl` 사용.
- **플랫폼**: React Native (Expo) + NativeWind(Tailwind). 웹 전용 HTML/div·span·shadcn 등 사용 금지.

---

## 만들어야 할 페이지 수 (구분)

| 구분 | 페이지 수 | 페이지 |
|------|-----------|--------|
| 인증 | 2 | 로그인, 회원가입 |
| 온보딩 | 1 | 온보딩 슬라이드 |
| 로딩/스플래시 | 1 | 인덱스(로딩) |
| 메인 탭 | 6 | Inbox, Schedule, Calendar, Notes, Review, Profile |
| **합계** | **10** | |

※ 실제로 “화면 디자인”이 필요한 건 **인증 2 + 온보딩 1 + 탭 6 = 9개**이고, 인덱스는 로딩 스피너만 있는 단순 화면이라 1개로 세거나 생략 가능.

## 1. 인증 (Auth) — 2페이지

### 1-1. 로그인 (Sign In)

- **경로**: `/sign-in`
- **역할**: 이메일/비밀번호 로그인, 회원가입 이동.
- **핵심 UI**
  - 상단: 앱 아이콘(또는 로고) + "다시 오신 걸 환영해요" + 한 줄 설명.
  - 카드 1개: 이메일 입력, 비밀번호 입력, [로그인] 버튼.
  - 에러 시: 카드 안에 빨간 배경 박스로 에러 메시지.
  - 하단: "계정이 없으신가요?" + [계정 만들기] 텍스트 버튼.
- **컨셉**
  - 배경 `bg-slate-50` / `dark:bg-zinc-950`.
  - 폼 전체를 하나의 흰색(다크 시 zinc-900) 카드로, 둥근 모서리·얇은 테두리·약한 그림자.
  - 타이틀은 크고 굵게(`text-3xl font-extrabold`), 설명은 본문 스타일.
  - CTA는 primary 버튼 하나로 강조.

---

### 1-2. 회원가입 (Sign Up)

- **경로**: `/sign-up`
- **역할**: 이메일/비밀번호 가입, 로그인으로 돌아가기.
- **핵심 UI**
  - 상단: 앱 아이콘 + "계정 만들기" + 한 줄 설명.
  - 카드 1개: 이메일, 비밀번호(6자 이상 안내), [가입하기] 버튼.
  - 하단: "이미 계정이 있으신가요?" + [로그인] 텍스트 버튼.
- **컨셉**
  - 로그인과 동일한 톤: slate 배경, 한 장의 카드, 명확한 타이포 위계.

---

## 2. 온보딩 — 1페이지

### 2-1. 온보딩 (Onboarding)

- **경로**: `/onboarding`
- **역할**: 첫 사용자에게 Inbox → Schedule → Calendar/Notes/Review 흐름 소개 후 메인으로 이동.
- **핵심 UI**
  - 가로 스와이프 슬라이드 3장.
    - 1: "Dump your thoughts" — Inbox로 생각 적기/말하기.
    - 2: "AI builds your week" — Schedule로 주간 계획 생성.
    - 3: "Calendar, notes, review" — 캘린더·노트·주간 리뷰.
  - 각 슬라이드: 이모지 + 제목 + 본문 1~2줄.
  - 하단: 페이지 인디케이터(점 3개) + [Skip] / [Next] / [Start].
- **컨셉**
  - 심플·미니멀. 배경 slate, 텍스트만으로도 읽기 쉽게.
  - 인디케이터는 선택된 것만 길게/강조 색, 나머지는 작은 원.

---

## 3. 로딩/인덱스 — 1페이지 (선택)

### 3-1. 인덱스/로딩 (Index / Loading)

- **경로**: `/` (리다이렉트 전 잠깐 보일 수 있음)
- **역할**: 세션·온보딩 확인 후 sign-in / onboarding / (tabs)/inbox로 이동. 로딩 중에만 표시.
- **핵심 UI**
  - 화면 중앙: 스피너(또는 로고) + "Loading..." 같은 짧은 문구.
- **컨셉**
  - 배경 `bg-slate-50` / `dark:bg-zinc-950`, 최소한의 요소만. v0에서 별도로 만들지 않고 “로딩 상태” 컴포넌트 1개로 처리해도 됨.

---

## 4. 메인 탭 — 6페이지

### 4-1. Inbox (브레인 덤프 / 구조화)

- **경로**: `/(tabs)/inbox`
- **역할**: 생각·할 일을 빠르게 적고, 한 줄 추가 또는 큰 입력창 + [항목 추가]. 나중에 [구조화하기]로 일정·노트로 넘김. 음성 입력 버튼.
- **핵심 UI**
  - 헤더: "브레인 덤프" / "구조화하기" 타이틀 + [구조화하기] 버튼(우측).
  - 카드 1개(입력 카드):
    - 한 줄 입력 + [항목 추가] 버튼.
    - 큰 멀티라인 입력창("자유롭게 입력하세요...").
    - [항목 추가] [음성] 버튼.
  - 섹션: "최근 항목 (N)".
  - 리스트: 항목 카드(텍스트, 파싱됨/저장됨 뱃지, [구조화] [노트로 저장] 등). 없으면 빈 상태: "아직 항목이 없어요" + [일정 만들기 →].
- **컨셉**
  - 배경 slate, 헤더는 흰색(zinc-900) 바에 하단 보더.
  - 입력 영역은 한 카드에 몰아서, 큰 입력창은 `rounded-2xl`, `min-h-[160px]` 느낌.
  - 항목은 카드 단위로, 라벨·버튼은 양끝 정렬.

---

### 4-2. Schedule (일정 생성)

- **경로**: `/(tabs)/schedule`
- **역할**: 목표/하고 싶은 일을 입력하면 AI가 주간 일정(활동 목록 + 시간) 생성. [저장] [캘린더에 추가] [새로 만들기].
- **핵심 UI**
  - 페이지 타이틀 + 짧은 설명.
  - 카드 1개: "어떤 일정을 만들고 싶나요?" + 큰 텍스트 영역 + 짧은 팁 문구.
  - [일정 생성] 버튼(풀폭).
  - 로딩 시: 스켈레톤 카드.
  - 에러 시: 빨간 톤 카드 + 에러 메시지 + [다시 시도].
  - 결과 시: 카드 안에 활동 리스트(이름, 주당 시간, 추천 시간대), 총 시간·가능 여부, 제안 문구, [저장] [캘린더에 추가] [새로 만들기].
- **컨셉**
  - 슬레이트 배경, 모든 블록 카드화. 결과 영역도 한 카드에 요약.

---

### 4-3. Calendar (캘린더)

- **경로**: `/(tabs)/calendar`
- **역할**: 월별 달력 + 주간 타임테이블(0~24시). 이벤트 목록, [추가] [Notes] [Import] 등.
- **핵심 UI**
  - 헤더: "Calendar" + [Notes] [Import(조건부)] [+ Add].
  - 카드 1개(캘린더 카드):
    - 월 네비: [‹] "2025년 2월" [›], [Month] [Timetable] 토글.
    - Month 뷰: 요일 헤더(S M T W T F S) + 6×7 그리드.
    - Timetable 뷰: 요일 열 + 0~23시 행, 셀에 짧은 이벤트 표시. 세로 스크롤.
  - 아래: "선택한 날 · Events" 섹션. 이벤트 카드 리스트(제목, 시간, 설명, [Edit] [Delete]). 없으면 빈 상태 + [일정 만들기].
- **컨셉**
  - 캘린더 블록은 흰색(zinc-900) 카드, 테두리·그림자. 이벤트 목록도 카드 단위.

---

### 4-4. Notes (노트)

- **경로**: `/(tabs)/notes`
- **역할**: 노트 목록(검색) + 선택 시 에디터. 태그, 일정 연동, AI 요약/제안.
- **핵심 UI**
  - 레이아웃: 좌측(또는 상단) 목록 / 우측(또는 전체) 에디터.
  - 목록 영역: "노트" 타이틀 + [새 노트] + 검색 입력창 + 노트 리스트(제목, 미리보기, 태그, 연동 일정 수, 수정일). 선택 시 강조.
  - 에디터 영역: 수정일, [취소] [저장], 제목 입력, 본문 텍스트 영역, 태그, 연동 일정, AI 인사이트(버튼 + 결과 카드). 노트 없을 때: "노트를 선택하거나 새로 만드세요" 빈 상태.
- **컨셉**
  - 배경 slate, 목록·에디터 영역은 흰색(zinc-900) 카드 느낌. 리스트 항목은 터치 영역 넉넉히.

---

### 4-5. Review (주간 리뷰)

- **경로**: `/(tabs)/review`
- **역할**: 주별 통계(달성률, 시간, 이벤트, 노트 수) + AI 인사이트 생성, [Copy] [Share] [Export].
- **핵심 UI**
  - 헤더: "주간 리뷰" 타이틀 + [Copy] [Share] [Export] [Generate]. 아래 주 선택: [Prev] "2/9–2/15" [Next].
  - 카드 1: "Achievement Overview" — 큰 퍼센트(달성률) + 뱃지(Excellent/Good/Keep Going).
  - 카드 2~4: "Key Metrics" — 시간(실제/계획), 이벤트(완료/전체), 노트 생성 수. 각각 라벨·숫자·뱃지 양끝 정렬.
  - 카드 5: "AI Insights" — [Generate] 후 인사이트 리스트(카드 형태). 없으면 안내 문구.
- **컨셉**
  - 숫자·퍼센트는 크고 굵게, 메트릭은 카드별로 구분. AI 인사이트는 회색 배경 작은 카드로 구분.

---

### 4-6. Profile (프로필 / 설정)

- **경로**: `/(tabs)/profile`
- **역할**: 계정 정보, 다크 모드, 언어, Pro 업그레이드, AI 사용량, 로그아웃.
- **핵심 UI**
  - 페이지 타이틀 "프로필"(또는 설정).
  - 카드 1: 아바타, 이름(이메일 아이디), 이메일, 플랜 뱃지(무료/Pro).
  - 카드 2: "다크 모드" + 상태(켜짐/꺼짐) + 스위치. 양끝 정렬.
  - 카드 3: "언어" + [한글] [English] 버튼.
  - 카드 4: "Pro 업그레이드" + 설명 + [업그레이드] [구매 복원](무료일 때만).
  - 카드 5: "이번 달 AI 사용량" — 일정 생성, 노트 파싱, AI 인사이트 각각 "N / 한도" 행. 양끝 정렬.
  - 하단: [로그아웃] 풀폭 버튼(위험 스타일).
- **컨셉**
  - 모든 설정을 카드로 묶고, 라벨·값·스위치/버튼은 `justify-between`. 이모지 없이 텍스트만으로 정돈.

## v0.dev용 한 줄 요약 (복사용)

- **로그인**: 토스 스타일 로그인 화면, 슬레이트 배경, 흰 카드에 이메일·비밀번호·로그인 버튼, 하단 계정 만들기 링크.
- **회원가입**: 토스 스타일 회원가입, 슬레이트 배경, 흰 카드에 이메일·비밀번호·가입 버튼, 하단 로그인 링크.
- **온보딩**: 3장 가로 스와이프, 이모지+제목+본문, 하단 인디케이터+Skip/Next/Start, 슬레이트 배경.
- **Inbox**: 브레인 덤프, 헤더+입력 카드(한 줄+큰 입력창+항목 추가·음성 버튼)+최근 항목 리스트/빈 상태, 토스·노션 톤.
- **Schedule**: AI 일정 생성, 타이틀+설명+입력 카드+생성 버튼+결과 카드(활동 리스트·저장·캘린더 추가), 슬레이트·카드 스타일.
- **Calendar**: 월/주 타임테이블 캘린더 카드+선택일 이벤트 카드 리스트, 헤더에 Notes·Import·Add, 토스·노션 톤.
- **Notes**: 좌측 노트 목록(검색·새 노트)+우측 에디터(제목·본문·태그·AI 인사이트), 카드 스타일.
- **Review**: 주간 리뷰, 헤더+주 선택+달성률 카드+메트릭 카드 3개+AI 인사이트 카드, 숫자 강조.
- **Profile**: 프로필·설정, 계정 카드·다크모드·언어·Pro·사용량 카드+로그아웃 버튼, 양끝 정렬 리스트 스타일.

## 참고

- **공통**: React Native (Expo) + Tailwind(NativeWind). `View`, `Text`, `ScrollView`, `Pressable`, `TextInput` 등만 사용.
- **다크 모드**: 위 컨셉의 `dark:` 클래스대로 모든 페이지 대응.
- **i18n**: 한글·영어 지원. v0에서는 한글 기준으로 와이어프레임 잡고, 영문은 나중에 치환해도 됨.
