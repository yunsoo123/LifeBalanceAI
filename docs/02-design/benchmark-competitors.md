# 경쟁사 벤치마크: Notion, Todoist, TickTick

**목적**: Mendly가 to-do·생산성 앱 경쟁에서 살아남으려면 어디를 맞춰야 하고, 어디로 차별화할지 정리.

**현실 인정**: 현재 Mendly의 기능·디자인은 **목업 수준에도 미치지 못함**. 이 문서는 “어디가 비어 있는지”와 “어디를 먼저 채울지”를 기준으로 삼기 위함.

---

## 1. Notion

### 핵심 UX·기능 (벤치마크 포인트)

| 영역 | Notion이 하는 것 | Mendly 현재 |
|------|------------------|-------------|
| **블록 기반** | 모든 콘텐츠가 블록(텍스트, 표, 이미지, 토글…). 끌어다 놓기, 타입 변환. | 단순 텍스트 입력 + “항목 추가” 버튼. 블록/재배열 없음. |
| **Quick Capture** | 모바일에서 바로 페이지 생성·검색·특정 문서 열기. 진입 장벽 최소. | Inbox 한 화면에 큰 입력창 하나. “빠른 캡처” 느낌은 있으나, 저장 후 흐름(→ 노트/일정)이 시각적으로 이어지지 않음. |
| **AI** | 블록/페이지 전체에 걸친 수평 레이어. 문맥에 맞는 도구가 필요할 때 등장. | Schedule/Inbox/Notes/Review 각각 별도 “Parse/Generate/Enhance” 버튼. AI가 붙어 있는 느낌이 약함. |
| **빈 화면·온보딩** | 템플릿, 가이드 블록, “시작하기” 흐름이 명확. | Empty State 텍스트 + 버튼 수준. 처음 쓰는 사람이 “뭘 해야 하지?” 느낌. |
| **디자인 완성도** | 타이포·여백·색 대비·일관된 컴포넌트. “프로덕트” 느낌. | 디자인 시스템 도입했으나, 전반적으로 아직 목업 이하. |

### Mendly가 배울 점

- **캡처 → 변환 → 배치**가 한 흐름으로 보이게 (Inbox → 구조화 → Schedule/Calendar/Notes로 가는 시각적 연결).
- **빈 상태 = 첫 사용 가이드** (한두 번만에 끝나는 미니 온보딩).
- **AI는 “버튼 하나”가 아니라, 입력/선택 시 자연스럽게 등장하는 레이어**.

## 2. Todoist

### 핵심 UX·기능 (벤치마크 포인트)

| 영역 | Todoist가 하는 것 | Mendly 현재 |
|------|-------------------|-------------|
| **Quick Add** | 한 줄에 작업 + 날짜 + 라벨 + 반복까지. “가장 빠르게 태스크 추가”에 올인. | Inbox는 여러 줄 텍스트. “항목 추가” 후 Parse로 구조화. 단계가 나뉘어 있어 빠른 캡처보다는 “덤프”에 가깝다. |
| **자연어** | “다음 목요일 오후 3시에 팀 미팅”, “매주 월요일” 등 입력만으로 날짜/반복 해석. | 날짜/반복은 Schedule·Calendar에서 별도 입력. Inbox 텍스트에서 자동 추출 없음. |
| **음성** | Ramble: 말하면 실시간으로 태스크로 정리. | 음성 녹음 → Whisper 파싱. 괜찮으나 “말하는 즉시 리스트로 보인다” 수준은 아님. |
| **UI 철학** | 미니멀. 자주 쓰는 것만 노출, 나머지는 메뉴 뒤. | 버튼·카드가 많고, 위계가 약해 “한 화면에 다 넣은” 느낌. |
| **통합** | 80+ 연동. | 없음. |

### Mendly가 배울 점

- **Inbox = Quick Add**: 한 줄 입력만으로 “할 일 + (선택) 언제/반복”이 파싱되게. 지금의 “여러 줄 덤프 → Parse 버튼”을 “한 줄 추가 = 곧바로 구조화된 항목”으로 단축.
- **자연어 날짜/시간**: Inbox나 Schedule 입력에서 “내일”, “다음 주 월요일” 등 해석.
- **화면당 한 가지 일**: 할 일 추가 화면은 “추가”에만 집중, 나머지는 다른 탭/모달로.

## 3. TickTick

### 핵심 UX·기능 (벤치마크 포인트)

| 영역 | TickTick가 하는 것 | Mendly 현재 |
|------|---------------------|-------------|
| **올인원** | 할 일 + 캘린더 + 습관 + Pomodoro + Eisenhower 매트릭스. | 할 일(Inbox)→Schedule→Calendar→Notes→Review. 기능은 있는데 “하나의 흐름”으로 묶인 느낌이 약함. |
| **캘린더** | 드래그 앤 드롭, 여러 뷰, 외부 캘린더 연동. | 주간 뷰 + 이벤트 CRUD. 드래그로 시간 이동, 구글 캘린더 연동 등 없음. |
| **습관·타이머** | 습관 트래커, Pomodoro가 앱 안에. | 없음. (Review는 “주간 회고”에 가깝고, 습관/포모도로는 아님.) |
| **디자인** | 기능이 많아도 정리된 UI. Todoist보다 옵션이 많이 보이지만 “지저분하다”보다 “다 있다”에 가깝게. | 기능 수는 있는데, 정돈·위계 부족으로 “목업 미만” 인상. |

### Mendly가 배울 점

- **캘린더와 할 일의 연결**: Inbox/Schedule에서 만든 게 캘린더에 어떻게 반영되는지 시각적으로 명확하게.
- **한 앱 안에서 “일정·할 일·회고”가 한 흐름**: 탭만 많고 흐름이 약한 상태를 줄이기.

## 4. Mendly vs 경쟁사 요약

| 기준 | Notion | Todoist | TickTick | **Mendly 현재** |
|------|--------|---------|----------|------------------|
| **캡처 속도** | 빠름 (블록·Quick Capture) | 매우 빠름 (Quick Add·자연어) | 빠름 | 보통 (덤프 → Parse 2단계) |
| **자연어** | 제한적 | 강함 (날짜·반복·라벨) | 있음 | 없음 |
| **디자인 완성도** | 높음 | 높음 (미니멀) | 높음 | **낮음 (목업 미만)** |
| **AI 통합** | 수평 레이어 | 작업 제안·생성 | 제한적 | 있으나 “버튼 위주” |
| **캘린더·일정** | DB/뷰 수준 | 기본 | 강함 (드래그·연동) | 주간 뷰·CRUD만 |
| **차별화** | 블록·올인원 워크스페이스 | 속도·자연어 | 올인원·습관·타이머 | **AI 주간 일정·브레인 덤프→구조화** (아이디어는 있으나 실행이 미흡) |

## 5. 결론: “목업 수준도 안 된다”를 어떻게 넘을지

### 5.1 디자인 (목업 수준 도달)

- **한 화면 = 한 가지 목적**: Inbox는 “캡처”, Schedule은 “입력 + 결과”, Calendar는 “보기 + 드래그”.
- **타이포·여백·컴포넌트 일관**: 이미 도입한 LAYOUT/design-system을 **전 화면**에 적용하고, 빈 상태·로딩·에러 상태까지 하나의 톤으로.
- **빈 화면 = 첫 사용 가이드**: “항목이 없어요”만이 아니라 “여기 한 줄 적고 → 구조화하기 누르면 → 일정/노트로 넘어갑니다” 수준의 한두 문장 + (선택) 1회성 미니 투어.

### 5.2 기능 (경쟁 앱과 비교해 “맞춤”)

1. **Quick Add + 자연어 (Todoist 수준 목표)**  
   Inbox에서 한 줄 입력 시 “내일”, “매주 월요일” 등 날짜/반복 추출 → 구조화된 항목으로 바로 추가. Parse 버튼은 “여러 줄 덤프”용으로 유지.
2. **캡처 → 일정 연결 시각화**  
   Inbox 항목이 “Schedule에 반영됨 / Calendar에 반영됨 / Notes에 저장됨”이 한 번에 보이게 (뱃지, 링크, 또는 짧은 플로우).
3. **AI를 “버튼” 밖으로**  
   입력창 포커스 시 “AI로 구조화”, “일정 제안” 등이 제안으로 뜨거나, 결과 카드 옆에 “다시 생성”/“노트로 저장”이 자연스럽게 붙게.

### 5.3 Mendly만의 차별화 (유지·강화)

- **브레인 덤프 → AI 주간 일정**: “생각을 쏟아넣으면 AI가 주간 계획을 잡아준다”는 흐름을 UI·카피·플로우로 명확히.
- **주간 리뷰 + 인사이트**: Todoist/TickTick에 없는 “한 주를 돌아보고 AI가 짧은 인사이트”를 주는 부분을 강조.
