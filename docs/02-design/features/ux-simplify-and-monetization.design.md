## Overview

실사용 피드백(시간 확인 어려움, Inbox 2단계, 타임테이블 찾기 어려움, 메모장 같은 UI)을 해소하기 위해 **시간 블록**, **Inbox CTA 단일화**, **타임테이블 노출**, **카드 스타일**을 구체 스펙으로 정의한다. 구현 시 이 문서를 기준으로 do 단계 진행.

## 1. 시간 블록 (Time block) — 구체 스펙

### 1.1 목표

- 일정/활동에서 **“몇 시 ~ 몇 시”**가 **한눈에** 보이도록 시간 전용 블록을 두고, 타이포·색으로 강조.

### 1.2 적용 위치

| 위치 | 파일 | 현재 | 변경 후 |
|------|------|------|---------|
| 캘린더: 선택일 이벤트 카드 | `app/(tabs)/calendar.tsx` | 제목 아래 `text-[12px] text-gray-400` 로 시간 | **시간 블록**을 카드 상단 또는 색 바 왼쪽에 배치 |
| 스케줄: 활동 리스트 각 행 | `app/(tabs)/schedule.tsx` | `hoursPerWeek`, `optimalTime`, overrides는 작은 글씨 | **시간 블록**으로 “요일 · HH:MM–HH:MM” 또는 “요일 HH:MM 시작” 명시 |

### 1.3 캘린더 이벤트 카드 레이아웃 (시간 블록)

- **구조 (위에서 아래)**  
  1. **시간 블록** (필수): 한 줄. `formatTime(start)` + ` – ` + `formatTime(end)`.  
     - 스타일: `text-[15px] font-semibold`, 색상 `colors.gray[700]` (라이트) / `colors.gray[300]` (다크).  
     - 선택: 시간 앞에 작은 시계 아이콘 또는 뱃지 배경(`rounded-lg px-2 py-0.5`)으로 시간만 감싸서 블록화.  
  2. **제목**: 기존 유지. `text-[15px]` 또는 `text-base` `font-semibold`.  
  3. **설명**(있을 때): `text-[13px]` 보조 색, `mt-1`.  
  4. **액션**(수정/삭제): 기존 유지, 터치 영역 44dp.

- **최소 구현**: 시간 라인만 **`text-[15px] font-semibold`** 로 키우고, 제목 위에 배치(또는 색 바와 같은 행에 시간 오른쪽). 기존 `text-[12px] text-gray-400` 제거.

### 1.4 스케줄 활동 리스트 시간 블록

- 각 활동 행에 **“요일 · 시작시간”** 또는 **“요일 HH:MM”** 을 **한 줄로 강조**.  
- **표시 규칙**: `overrides[index]` 가 있으면 해당 요일·시작시간, 없으면 기본 슬롯(또는 “요일 미배정” 후 시간 변경 버튼).  
- 스타일: `text-[14px] font-semibold`, 라이트/다크 각각 가독성 있는 색.  
- 배치: 활동명·시간 변경 버튼과 같은 행에서 **시간 정보를 왼쪽 또는 중앙**에 한 블록으로 묶어 표시.

## 2. Inbox CTA — 구체 스펙

### 2.1 목표

- **한 흐름 한 CTA**: 입력 → (선택) 구조화 → **“캘린더에 추가”** 한 번으로 일정 생성 완결.  
- “구조화하기”와 “자동 일정”이 같은 영역에 두 개 있는 현재 구조 정리.

### 2.2 현재 구조 (참고)

- 한 줄 입력 + **“일정으로 구조화”** (parse) → `parsedScheduleEvents` 리스트 표시 → **“자동 일정”** (runAutoSchedule) 으로 캘린더 insert.  
- 별도로 **“구조화하기”** / **“자동 일정”** 버튼이 하단에 또 있음(parseAllEntries 등). 중복·혼동 유발.

### 2.3 변경 후 구조

- **상단 입력 영역**  
  - 한 줄 입력 유지.  
  - **주 CTA 1개**:  
    - **옵션 A (권장)**: 라벨 **“일정으로 구조화하고 캘린더에 추가”** (또는 “한 번에 캘린더에 넣기”). 클릭 시 **파싱 + 자동 배치를 연속 실행** (parse API → 성공 시 즉시 runAutoSchedule 로직 호출). 로딩 중에는 하나의 스피너로 표시.  
    - **옵션 B**: 기존처럼 파싱만 먼저 하되, 파싱 결과 블록 안에는 **버튼 1개만** — **“캘린더에 N개 추가”** (N = parsedScheduleEvents.length). “자동 일정” 문구 제거하고 이 라벨로 통일.

- **하단 버튼 정리**  
  - **“구조화하기”** / **“자동 일정”** 둘 다 있는 행 제거 또는 **하나로 통합**.  
  - 예: **“전체 구조화”** (기존 parseAllEntries) 만 두거나, “캘린더에 추가”가 이미 위에 있으면 하단은 **“노트로 저장”** 등 다른 액션만 유지.

### 2.4 라벨·메시지

- 성공 시: “캘린더에 N개 일정이 추가되었어요.” (기존 유지 가능).  
- 파싱 실패/배치 실패 시: 기존 Alert 유지, 문구만 짧고 명확하게.

### 2.5 구현 순서

1. **옵션 B** 먼저: 파싱 결과 블록에서 “자동 일정” → **“캘린더에 N개 추가”** 로 라벨 변경, 하단 중복 버튼 제거 또는 하나로 통합.  
2. (선택) **옵션 A**: 한 번에 “구조화 + 캘린더 추가” 플로우로 확장.

## 3. 타임테이블 노출 — 구체 스펙

### 3.1 목표

- Calendar 탭에서 **주간 시간표(타임테이블)** 를 **바로 찾을 수 있게** 한다.

### 3.2 방식 (택 1 또는 병행)

**방식 A: 기본 뷰를 Timetable로**

- `app/(tabs)/calendar.tsx` 에서 `viewMode` 초기값을 `'timetable'` 로 변경.  
- `useState<'month' | 'timetable'>('timetable')`.  
- 월별 그리드를 보려면 사용자가 “Month” 토글을 누르면 됨.

**방식 B: 뷰 전환 UI 강조 (기본은 Month 유지)**

- Month/Timetable 토글을 **헤더 바로 아래** 또는 **달력 위**에 두고, 라벨을 **“월간” / “이번 주”** (또는 “주간”) 로 변경.  
- “이번 주” 쪽을 **조금 더 눈에 띄게** (아이콘 📅 vs 🗓 등) 표시.

**권장**: **방식 A** 적용. “주간 시간표”를 찾는 사용자 비율이 높다고 가정할 때 기본이 Timetable이면 발견성이 가장 좋음.

### 3.3 부가 사항

- 스케줄에서 “캘린더에 등록하기” 후 이동 시 `focusWeek` 파라미터로 해당 주·타임테이블 노출하는 기존 동작 유지.  
- (선택) 첫 진입 시 `viewMode === 'month'` 인 경우, “이번 주 보기” 링크 한 번 표시 후 타임테이블로 이동하는 안내는 별도 이슈로 둠.

## 4. 카드 스타일 — 구체 스펙

### 4.1 목표

- 이벤트·활동·리스트가 **“글자 나열”이 아닌 블록/카드 위계**로 보이도록.  
- v0·design-system 과 일치: 카드로 묶고, 시간/제목/메타를 시각적으로 분리.

### 4.2 공통 카드 토큰 (v0 + design-system)

- **컨테이너**: `rounded-2xl`, `p-5` (또는 `p-4` 최소), `shadow-sm`, `border`.  
- **배경**: 라이트 `bg-white` / `#fff`, 다크 `bg-zinc-900` / `#252631` (design-system `surface.darkCard` 계열).  
- **테두리**: 라이트 `border-gray-200` / 다크 `border-zinc-700`.  
- **내부 간격**: 섹션 간 `gap-3` 또는 `mb-3`, 블록 내 `gap-2`.

### 4.3 캘린더 이벤트 카드 구조

- **레이아웃**  
  - 왼쪽: **색 바** (세로 `w-1`, `rounded-full`, `event.color`), 높이 카드와 동일 또는 최소 40px.  
  - 오른쪽:  
    - **블록 1 – 시간**: `text-[15px] font-semibold`, 위에서 정의한 시간 블록.  
    - **블록 2 – 제목**: `text-base font-semibold` (또는 `text-[15px]`).  
    - **블록 3 – 설명**(있을 때): `text-[13px]` 보조 색.  
  - 오른쪽 끝: 수정/삭제 아이콘, `min-h-[44px] min-w-[44px]`.

- **패딩**: 카드 전체 `p-4` 이상, 색 바와 텍스트 사이 `gap-3`.

### 4.4 스케줄 활동 카드(리스트 행) 구조

- **행 구성**: 아이콘 영역 | **이름** | **시간 블록(요일 · HH:MM)** | 시간 변경 버튼.  
- **시간 블록**: 같은 행 안에서 `text-[14px] font-semibold` 로 한 덩어리.  
- 카드/리스트 컨테이너: 기존 `rounded-2xl`, `border`, `surface.resultCard` 등 유지, 내부만 위 블록 순서로 정리.

### 4.5 Inbox 입력·결과 카드

- **입력 카드**: 한 줄 입력 + 멀티라인 입력을 하나의 카드로. `rounded-2xl`, `p-5`, `border`, v0 입력 스타일.  
- **파싱 결과 블록**: “파싱된 일정” 리스트 + **“캘린더에 N개 추가”** 버튼 하나. 리스트 항목은 `text-[14px]`, 구분선 또는 `gap-2` 로 나열.  
- **placed events 미리보기**: 기존 “타임테이블 미리보기” 카드 유지, 카드 스타일은 위 공통 토큰 적용.

### 4.6 빈 상태·성공 피드백

- **빈 상태**: 아이콘 + 제목 + 짧은 설명 + (선택) CTA. `text-base` / `text-[15px]` 본문, 보조 `text-[13px]`.  
- **성공**: Alert 또는 토스트 “캘린더에 N개 추가되었어요.” 문구 유지, 가능하면 2초 내 자동 닫힘 또는 확인 버튼 하나.
