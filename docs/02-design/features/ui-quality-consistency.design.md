# UI/UX 전 페이지 퀄리티 일관화 — Design (구조·인터랙션 우선)

**Date**: 2025-02-25  
**Author**: PDCA + ui-design  
**Project**: Mendly (LifeBalanceAI)  
**Related**: [ui-quality-consistency.plan.md](../../01-plan/features/ui-quality-consistency.plan.md)

---

## 핵심 원칙

- **색상만 맞추는 것이 아니다.** 색상은 보조. **목표는 "메모장 같다"를 벗어나 카톡·노션·틱틱처럼 구조·인터랙션·계층이 있는 "앱다운" 디자인.**
- 아래 **1. Product feel**을 1순위로 적용하고, **2~10**의 시각 스펙(색·타이포·카드 등)은 그 구조를 지원하는 수준으로 적용.

---

## 1. Product feel: 카톡 · 노션 · 틱틱 (1순위)

### 1.1 카톡처럼 — 대화·입력·반응

| 적용 화면 | 구조·인터랙션 |
|-----------|----------------|
| **Schedule** | • **대화 스레드**: 사용자 말풍선 / AI 말풍선이 명확히 구분되고 시간순 스크롤.<br>• **하단 고정 입력창**: 플레이스홀더 + 전송/마이크, 터치 시 키보드 올라오고 스크롤이 말풍선 쪽으로.<br>• **결과는 "카드 한 장"**: 스케줄 요약·진행바·CTA가 **한 블록**으로 묶여 채팅 안에 삽입되는 느낌(메모장 나열 아님). |
| **Inbox** | • **빠른 입력**: "QUICK NOTE"처럼 **한 줄 입력 + 바로 액션**(Auto-Schedule)이 한 블록. 메모장처럼 여러 줄만 나열되지 않게.<br>• **결과 피드백**: "3 Recurring Events Created"처럼 **한 문장으로 결과**가 보이게. |

### 1.2 노션처럼 — 블록·계층·카드

| 적용 화면 | 구조·인터랙션 |
|-----------|----------------|
| **Notes** | • **블록 단위**: 제목 블록 / 본문 블록 / 체크리스트 블록 / 태그 블록이 **시각적으로 구분**되고, 블록 단위로 터치·편집 가능한 느낌.<br>• **계층**: 제목 > 섹션 > 본문 > 캡션 순서가 한눈에 들어오게. |
| **Review** | • **뷰 단위**: "WEEKLY REPORT" 캡션 → 제목 → **메트릭 카드 2열** → **목표 vs 실제 카드**(행 단위: 아이콘 + 이름 + 수치 + 진행바) → 하단 배너. 각각 **카드/블록**으로 묶여 "DB 뷰"처럼 보이게.<br>• **숫자·진행바**가 블록 안에 들어가 있어 "한 덩어리"로 인지되게. |
| **전체** | • 리스트가 있으면 **항목 = 카드 한 장** 또는 **행 한 줄**이 명확. 단순 텍스트 나열 지양. |

### 1.3 틱틱처럼 — 일정·태스크·진행

| 적용 화면 | 구조·인터랙션 |
|-----------|----------------|
| **Calendar** | • **일정 = 타임라인/그리드 위 블록**: 색·시간대가 보이고, 터치 시 상세/편집. "메모장 리스트"가 아니라 **캘린더 뷰**가 메인.<br>• **일정 추가**: 포스트잇/모달에서 **한 블록**으로 제목·시간·반복 등이 묶여 있음. |
| **Inbox → Schedule** | • 할 일/메모가 **"일정 블록"으로 전환**되는 흐름이 보이게(예: Auto-Schedule 후 "캘린더에 반영됨" + 미리보기 블록). |
| **Review** | • **목표 vs 실제**가 "태스크 진행"처럼: 행별 진행바, 달성/미달 색상. 틱틱의 리스트·필터 느낌을 참고해 **구조만 명확**하게. |

### 1.4 "메모장 같음" 제거 체크

- [ ] 모든 리스트가 **카드 또는 행 블록**으로 구분되어 있음(평면 텍스트 나열 아님).
- [ ] 채팅/입력 화면은 **말풍선 + 고정 입력창** 구조가 명확함.
- [ ] 결과/요약은 **한 블록(카드)** 안에 제목·수치·CTA가 같이 있음.
- [ ] 캘린더는 **그리드/타임라인 + 이벤트 블록**이 메인이고, 단순 리스트가 아님.

## 2. 색상 (다크 우선) — 구조를 지원하는 수준

| 용도 | 값 | 비고 |
|------|-----|------|
| 화면 배경 | `#1A1B27` | 기존 DARK_BG_COLOR 유지 |
| 카드 배경(다크) | `#252631` 또는 `#2E2E4A` | 결과/입력 카드. 기존 surface.darkCard, resultCard |
| 사용자 말풍선/CTA | `#6C63FF` ~ `#8B5CF6` | brand.primary 계열 통일 |
| AI 말풍선/보조 영역 | `#3A3A4A` ~ `#3f3f46` | chat.aiBubble, zinc-700 |
| 온라인/성공 | `#10B981` (emerald) | • Online, 체크, 긍정 지표 |
| 경고/미달 | `#EF4444` 또는 red-500 | 감소, Over Budget |
| 초과(보라) | `#8B5CF6` | 300% 등 과다 지표 |
| 본문 텍스트 | `#fafafa` / `#f3f4f6` | white / gray-100 |
| 보조 텍스트 | `#9ca3af` / `#6b7280` | gray-400 / gray-500 |
| 캡션/라벨 | `#6b7280`, 13px 또는 uppercase 작게 | QUICK NOTE, WEEKLY REPORT 스타일 |

- 라이트 모드: 기존 design-system 유지(배경 흰색/슬레이트, 텍스트 gray-900/600).

## 3. 타이포그래피

| 역할 | 크기 | 굵기 | 행간 | 용도 |
|------|------|------|------|------|
| 페이지 제목 | 20~24px | bold(700) | 1.2~1.3 | 2월 3주차 리포트, React Native Navigation |
| 섹션 제목 | 16~18px | semibold(600) | 1.3 | 목표 vs 실제, 활동 시간 분석 |
| 숫자(메트릭) | 24~36px | bold(700) | 1.2 | 42h, 78%, 112h |
| 본문 | 15~16px | normal(400) | 1.5~1.75 | 채팅, 카드 설명 |
| 캡션/라벨 | 12~13px | 500~600 | 1.3 | QUICK NOTE, WEEKLY REPORT, 라벨 |
| 버튼 라벨 | 15~16px | semibold(600) | - | 캘린더에 등록하기, Auto-Schedule |

- 폰트: 기존 Inter / 시스템 sans 유지.

---

## 4. 카드 (블록 단위 시각)

- **모서리**: 12px ~ 20px (rounded-xl ~ rounded-2xl).
- **패딩**: 16px ~ 24px (p-4 ~ p-6).
- **다크**: 배경 `#252631` 또는 `#2E2E4A`, 테두리 `rgba(63,63,70,0.6)` 또는 zinc-700.
- **라이트**: 배경 white, 테두리 gray-200.
- **그림자**: 카드가 떠 있는 느낌일 때 shadow-md ~ shadow-lg (참조의 결과 카드, 메트릭 카드).
- **역할**: "한 덩어리"를 표현해 **노션/틱틱 같은 블록·카드** 느낌을 주기 위함.

---

## 5. 헤더 (참조 1·2)

- **왼쪽**: 로고(원형 또는 아이콘) + 앱명 "LifeBalance AI" / "Mendly" (흰색, bold).
- **아래**: "• Online" 초록 점 + 텍스트 (13px, green).
- **오른쪽**: 메뉴(⋮) 또는 캘린더 아이콘. 터치 영역 44dp 이상.
- **구분선**: border-b, zinc-800 또는 gray-200.

---

## 6. 입력 영역 (카톡형 고정 입력바)

- **다크**: 배경 다크 그레이(zinc-800/900), placeholder 연한 회색(gray-400), 테두리 zinc-700.
- **플레이스홀더**: "메시지를 입력하세요...", "자유롭게 입력하세요..." 등 i18n.
- **하단 고정 입력바**: rounded, 좌측 + 아이콘, 우측 마이크(또는 전송). 높이 48~56px. **채팅 화면에서는 반드시 고정.**
- **Auto-Schedule / CTA 버튼**: 보라 배경, 흰 텍스트, rounded-xl, 아이콘(번개 등) 선택.

---

## 7. CTA 버튼

- **주요 액션**: 배경 brand.primary(#8B5CF6 또는 #6C63FF), 흰 텍스트, semibold, minHeight 48px, rounded-xl.
- **비활성**: gray-400 등으로 구분.
- **보조(취소/아웃라인)**: 테두리만 또는 연한 배경.

---

## 8. 진행 바 (목표 vs 실제·틱틱형)

- **트랙**: 배경 gray-700/zinc-700, 높이 6~8px, rounded-full.
- **채움**: 성공=green, 미달=red, 초과=purple. rounded-full.
- **레이블**: "112h / 168h Total", "8h / 20h (40%)" 등 숫자+단위 명확히. **카드/행 블록 안에** 배치.

---

## 9. 이벤트/활동 블록 (캘린더·리스트)

- **캘린더 그리드 블록**: 파랑/초록/보라 등 활동별 색, rounded, **패딩 충분히**(최소 8px 상하)해 2줄 텍스트가 겹치지 않게. **일정 = 블록**으로 인지되게.
- **목표 vs 실제 행**: 아이콘 + 이름 + "8h / 20h (40%)" + 진행 바. **한 행 = 한 블록** 느낌.

---

## 10. 채팅 말풍선 (카톡형)

- **사용자**: 배경 #6C63FF(또는 brand.primary), 흰 텍스트, rounded-xl, 좌/우 패딩 16px.
- **AI**: 배경 #3A3A4A(zinc-700), 텍스트 #f3f4f6, rounded-xl.
- **결과 카드**: 채팅 안에 **한 카드 블록**으로 삽입(말풍선과 시각적으로 구분되되, 대화 흐름 안에 있음).

## 11. 적용 순서 (Do 단계) — 구조 먼저, 그다음 시각

| 순서 | 화면 | 1순위 (구조·인터랙션) | 2순위 (시각) |
|------|------|------------------------|--------------|
| 1 | Schedule | 말풍선 구분 + 고정 입력창 + 결과를 **한 카드 블록**으로 | 헤더·색·진행바 스타일 |

| 6 | Profile | 설정 **카드/행 블록** 구분 | 배너·색 |
| 7 | 가이드·모달 | 섹션을 **블록/카드**로 구분 | 타이포·CTA |

## 12. design-system / layoutConstants 보강

- **design-system.ts**: Product feel(카톡·노션·틱틱)을 지원하도록 블록/카드/채팅 관련 토큰 정리. 색은 기존 유지하되 주석으로 "구조 1순위" 명시.
- **layoutConstants**: 블록 패딩, 카드 간격, 입력창 높이 등 **구조에 필요한 수치** 우선.
- **공통 컴포넌트**: ChatBubble, ResultCard, MetricCard, ProgressBar, EventBlock 등이 **블록 단위**를 명확히 하도록 사용.

## 13. 체크리스트 (구현 시)

**구조·인터랙션 (1순위)**  
- [ ] Schedule: 대화 스레드 + 고정 입력창 + 결과가 한 카드 블록.
- [ ] Inbox: 빠른 입력 + 결과 문장 + 그리드 상 이벤트 블록.
- [ ] Review: 메트릭 카드 2열 + 목표 vs 실제 행 블록 + 배너.
- [ ] Calendar: 그리드/타임라인 + 일정 블록, 추가 시 한 블록 모달.
- [ ] Notes: 제목/본문/체크/태그 블록 구분.
- [ ] 리스트/결과가 평면 텍스트 나열이 아님(카드 또는 행 블록).

**시각 (2순위)**  
- [ ] 다크 배경·카드 색, CTA 보라, 성공 초록 일관.
- [ ] 카드 radius 12~20px, padding 16~24px.
- [ ] 페이지 제목 20~24px bold, 메트릭 24~36px bold, 본문 15~16px.
- [ ] 헤더 로고+앱명+Online+메뉴, 적용 화면에서 일관.
