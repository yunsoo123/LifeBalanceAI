## Overview

리뷰 탭의 **진척률 라벨/정의**를 UI 스펙에 명시하고, **번잡한 레이아웃을 간소화**하여 한눈에 읽기 쉽고 행동 경로가 분명한 화면으로 개선한다. 지시 사항(진척률 정의·간소화) 외 **추가 디자인·기능 개선**을 포함한다.

---

## 1. 진척률 라벨/정의 (UI 스펙)

### 1.1 정의 (문서·UI 동일)

| 지표 | 정의 | 공식 | 표시 조건 |
|------|------|------|-----------|
| **계획 달성률** (시간 기준) | 계획한 총 시간 대비 실제 수행한 시간 비율 | `(actual_hours / planned_hours) × 100` (%) | `planned_hours > 0`일 때만 숫자 표시. `planned_hours === 0`이면 "—" 또는 "계획 없음". |
| **이벤트 완료율** | 해당 주 일정 중 완료 처리한 비율 | `(completed_events / total_events) × 100` (%) | `total_events > 0`일 때. 이미 "완료한 일정 N / M"으로 표시 중이므로, 필요 시 라벨을 "이벤트 완료율"로 명시. |

- **계획 달성률**을 화면에서 "진척률"과 혼용하지 않도록, **라벨은 하나로 통일**:  
  - **표시 라벨**: `계획 달성률` (ko) / `Plan achievement rate` (en).  
  - **부가 설명(툴팁/도움말)**: "계획한 시간 대비 실제 한 시간 비율이에요." (ko) / "Actual hours done vs planned hours." (en).  
- **이벤트 완료율**은 "완료한 일정 N / M" 옆에 작게 "(이벤트 완료율 XX%)"로 표시하거나, 동일 행에 라벨만 "이벤트 완료율"로 두고 값은 `N / M` 유지. (선택: 툴팁에 정의만 넣어도 됨.)

### 1.2 UI 스펙

- **MetricCard "계획 달성률"**
  - 라벨: `t.review`의 기존 키 사용 시 의미가 "계획 달성률"과 일치하는지 확인. 불일치 시 i18n에 `planAchievementRateLabel`, `planAchievementRateTooltip` 추가.
  - 툴팁/접근성: `accessibilityHint` 또는 화면 하단/설정에 "진척률이란?" 링크로 위 정의 문구 노출.
- **목표 vs 실제 카드 하단**
  - "완료한 일정 N / M" 행의 라벨: 현재 `eventsCompleted` 유지. 필요 시 서브텍스트에 "이벤트 완료율 = 완료한 일정 수 ÷ 전체 일정 수" 한 줄만 추가 (또는 툴팁).

## 2. 리뷰 탭 레이아웃 간소화

### 2.1 현재 문제

- **섹션이 과다**: WEEKLY REPORT 캡션, 제목, 공유 → 메트릭 2열 → Over Budget 배지(복수) → 목표 vs 실제 → 이번 주 일정 → AI Coach's Tip → New Routine Generated → AI 패턴 분석. 스크롤이 길고 정보 계층이 흐려짐.
- **Over Budget**가 여러 개일 때 배지가 세로로 반복되어 시선이 분산됨.
- **AI 관련 블록이 3개** (Coach's Tip, New Routine, 패턴 분석)로 분리되어 있어 "AI"가 반복되어 번잡함.
- **헤더 액션** (복사, 내보내기, 인사이트 생성)이 한 줄에 나열되어 작은 화면에서 답답함.

### 2.2 제안 구조 (단일 스크롤, 위→아래)

| 순서 | 블록 | 내용 | 비고 |
|------|------|------|------|
| 0 | **주 선택 + 액션** | ◀ 이전 주 | 이번 주 날짜 | 다음 주 ▶. 오른쪽: **⋮ 메뉴** (복사, 공유, 내보내기, 인사이트 생성). | 메인 액션을 메뉴로 묶어 헤더 단순화. |
| 1 | **요약 카드 1장** | "이번 주 요약" 타이틀. **총 몰입 시간** (32h / 40h 계획) + **계획 달성률** (80%) + 전주 대비 ↑/↓. Over Budget가 있으면 **한 줄**로 "목표 초과: 활동명 +Nh" 콤마 구분 또는 툴팁. | 메트릭 2열을 **한 카드**로 합치고, Over Budget는 카드 내 한 줄 또는 배지 1개로 요약. |
| 2 | **진척률 정의** | 계획 달성률 옆 (i) 아이콘 또는 작은 "도움말" 링크 → 탭 시 "계획한 시간 대비 실제 한 시간 비율이에요." 툴팁/모달. | §1.2 반영. |
| 3 | **목표 vs 실제** | 기존 GoalVsActualBar 유지. 카드 하단에 "완료한 일정 N / M" + (선택) "이벤트 완료율 XX%". 노트 개수는 동일 행 또는 다음 줄 한 줄. | 카드 1개, 내부만 정리. |
| 4 | **이번 주 일정** | 완료 토글 리스트. 없으면 "이번 주 일정이 없어요" 작은 문구만. | 기존 유지, 카드 1개. |
| 5 | **AI 인사이트 (통합)** | **하나의 카드**: (1) 다음 주 제안이 있으면 상단에 말풍선 + 수락/나중에. (2) 그 다음 "AI 코치 팁" 한 줄 또는 인사이트 1개. (3) "인사이트 생성" 버튼 + 생성된 인사이트 목록(접을 수 있으면 접기). | "AI Coach's Tip", "New Routine Generated", "AI 패턴 분석"을 **한 섹션**으로 묶고, 우선순위만 구분. |

### 2.3 시각적 단순화

- **WEEKLY REPORT** 캡션: 유지하되 상단 여백만 조정. 또는 "이번 주 리뷰"로 변경해 덜 딱딱하게.
- **카드 통일**: 모든 콘텐츠 카드 `rounded-[20px]`, `border`, `p-5`, 동일 배경/테두리 톤. 섹션 제목은 카드 상단 **굵은 1줄**로만.
- **Over Budget**: 요약 카드 안에 텍스트 한 줄 또는 배지 1개(여러 개면 "N개 목표 초과" + 탭 시 상세).
- **여백**: 섹션 간 `mb-5` 또는 `mb-6` 유지해 호흡 확보.

## 3. 추가 디자인·기능 개선

### 3.1 접근성·일관성

- 모든 터치 타겟 **최소 44×44dp**.
- "계획 달성률", "총 몰입 시간" 등 **라벨에 i18n 키** 사용, 한/영 전환 시 일관된 문구.
- 다크 모드: 기존 `theme === 'dark'` 분기 유지, 카드/텍스트 색상 design-system과 동일.

### 3.2 로딩·빈 상태

- **로딩**: 기존 Spinner + "로딩 중..." 유지.
- **로그아웃**: Plan 대로 가짜 숫자 제거, EmptyState + "로그인하면 주간 리뷰를 볼 수 있어요" + 로그인 버튼만 표시.
- **데이터 없음**: "이번 주 데이터가 없어요" + Schedule로 가기 CTA 유지.

### 3.3 기능 개선 (선택)

- **진척률 도움말**: 요약 카드의 "계획 달성률" 옆 (i) 터치 시 모달 또는 인라인 툴팁에 §1.1 정의 문구 표시.
- **이벤트 완료율**: 목표 vs 실제 카드 하단 "완료한 일정 N / M" 옆에 `(이벤트 완료율 XX%)` 작은 글씨로 표시 (total_events > 0일 때).
- **AI 섹션 접기**: 인사이트가 3개 이상이면 "N개 인사이트"만 보이고 펼치기 버튼으로 전체 표시.
