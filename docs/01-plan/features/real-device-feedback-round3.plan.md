## 2. 전제 (사실 vs 가정)

- **사실**: 실기기 테스트 후 위 9건 의견 확인. 4번은 **일정 생성 불가** 수준의 버그이므로 최우선 수정. 9번은 Quick 수정.
- **가정**:
  - 1·2·3·5·6·7·8은 기존 plan/design과 연계하거나 소규모 스펙으로 진행 가능.
  - 2번 "매주 O 요일"은 번잡하면 패스 → 우선 1번(마감일) 구현 후, 반복은 단순 옵션(매일/매주/매월)만 먼저 검토.

---

## 3. 접근 방식 (옵션)

### A) 4·9만 즉시 수정, 나머지는 방안만 정리

- **4**: 시간 피커 스크롤 오류 수정(모달 ScrollView `nestedScrollEnabled` 등) + 일정 반복은 design 후 Do.
- **9**: 노트/버튼 다크모드 글자색 수정(Button 또는 notes 헤더 인라인 색).
- **1~3, 5~8**: 본 plan에 방안만 정리하고, 사용자 확인 후 순차 design → Do.

### B) 4·9 수정 + 3·5·8 Quick까지 한 번에

- 4·9 동일.
- **3**: 일정 추가 모달 날짜 영역 탭 시 날짜 피커(모달 또는 인라인) 추가.
- **5**: AI 대화 말풍선 간격 증가.
- **8**: 리뷰 페이지 카드/박스 구분(LAYOUT 적용).

**권장**: **A**로 4·9 즉시 수정하고, 나머지는 본 문서로 방안 확정 후 단계별 진행. 3·5·8은 Quick으로 가능하면 다음에 진행.
