## Overview

캘린더 일정 추가 시 폰트 겹침, 스케줄 탭 스크롤·타임테이블 반영, 일정 드래그 드롭, 일정 추가 폼 UI/UX 등 5가지 문제를 수정한다.

---

## 1. 요청 해석 (Deep-think)

| # | 표면 요청 | 실제 의도 |
|---|-----------|------------|
| 1 | 캘린더에서 일정 추가 시 폰트끼리 겹쳐서 입력 불가 | 일정 추가 모달(또는 상단 헤더+모달) 레이아웃/텍스트 배치 버그로 라벨·입력이 겹침 → 수정 시 입력 가능·가독성 확보 |
| 2 | 스케줄에서 일정 생성 시 스크롤이 아래로 안 내려가서 내용이 잘림 | 스케줄 탭 ScrollView가 끝까지 스크롤되지 않음 → 결과 카드·활동 리스트까지 스크롤 가능해야 함 |
| 3 | 스케줄 일정이 캘린더에만 적용되고 타임테이블엔 안 됨 | DB에는 동일 `events` 테이블 사용. 타임테이블이 “보는 주”를 월 기준으로 해서 다른 주를 보여줄 수 있음 → 스케줄 주와 캘린더/타임테이블 “보는 주” 정렬 필요 |
| 4 | 일정 드래그 드롭 기능 개선 | 현재: 롱프레스로 선택 후 셀 탭으로 이동. 개선: 시각 피드백·제스처(드래그) 또는 UX 명확성 |
| 5 | UI/UX 박스 형태·사용자 편의 개선 | 일정 추가 폼을 박스/카드 형태로 구분하고, 레이블·여백·터치 영역 정리 |

---

## 2. 전제 조건

- **P1**: 일정 추가는 캘린더 탭의 “Add Event” 모달에서 수행됨(`calendar.tsx`).
- **P2**: 스케줄 탭 “캘린더에 등록하기”는 `events` 테이블에 insert; 캘린더 탭은 동일 테이블을 `loadEvents()`로 조회(월 기준).
- **P3**: 타임테이블은 `getWeekDatesFor(viewingMonth)`로 “현재 보는 월의 1일이 속한 주”를 표시. 스케줄은 `getStartOfWeek()`로 “오늘 포함 주”에 이벤트 생성 → 보는 주가 다르면 타임테이블에 안 보일 수 있음.
- **P4**: 드래그 이동은 롱프레스로 `movingEventId` 설정 후 셀 탭으로 드롭하는 방식으로 이미 구현됨.

---

## 3. 알 것 / 모를 것

| 구분 | 내용 |
|------|------|
| **알 것** | Add Event 모달에 ScrollView 없음; 라벨이 없고 placeholder만 있음. 스케줄 ScrollView는 `scrollToEnd` 있으나 `contentContainerStyle`에 `flexGrow:1` 등으로 짧을 때만 채움. 타임테이블 주 계산은 `getWeekDatesFor(viewingMonth)`. |
| **모를 것** | 폰트 겹침의 정확한 원인(모달 내부 flex/lineHeight vs 헤더와 모달 동시 노출). “타임테이블에 안 보인다”가 “다른 주를 보고 있어서”인지, 아니면 데이터/필터 버그인지 사용자 환경에서의 재현 경로. |

---

## 4. 접근 (2가지)

**A. 최소 수정 (Quick fix)**  
- P1: 모달에 ScrollView + 라벨/입력 행 간격 확대, 키보드 대응.  
- P2: 스크롤 시 `scrollToEnd` 타이밍/조건 강화, 필요 시 `contentContainerStyle` 조정.  
- P3: “캘린더에 등록하기” 후 캘린더로 이동 시 `viewingMonth`를 “이번 주가 포함된 날”로 설정하고, 필요 시 `viewMode='timetable'`로 전환.  
- P4: 드롭 타겟 하이라이트 등 시각 피드백만 강화.  
- P5: 일정 추가 폼에 카드/박스 스타일과 여백 적용.

**B. 구조 개선 포함**  
- A에 더해, 일정 추가 폼을 별도 컴포넌트로 분리, 캘린더/타임테이블 “보는 주”를 “오늘 포함 주” 기본값으로 통일, 드래그를 제스처(pan)로 확장.

**권장**: **A 먼저** 적용으로 1~5번 현상 해소 후, 필요 시 B를 별도 PDCA로 진행.

## 5. 인수 조건 (Acceptance Criteria)

- [ ] **1** 캘린더에서 “일정 추가” 시 모달 내 라벨·입력이 겹치지 않고, 제목/설명/시작시간 입력 가능.
- [ ] **2** 스케줄 탭에서 일정 생성 후 결과 카드·활동 리스트까지 아래로 스크롤 가능.
- [ ] **3** 스케줄 “캘린더에 등록하기” 후 캘린더 탭에서 타임테이블 뷰로 이동 시, 해당 주에 일정이 보임(보는 주 정렬).
- [ ] **4** 일정 이동 시 드롭 타겟 시각 피드백이 명확함(및 필요 시 취소 안내).
- [ ] **5** 일정 추가 폼이 박스/카드 형태로 구분되고, 터치 영역·여백이 개선됨.

---

## 6. 범위 (Scope)

**In scope**  
- `app/(tabs)/calendar.tsx`: Add Event 모달 레이아웃(ScrollView, 라벨, 간격, 박스 스타일), 캘린더 이동 시 `viewingMonth`/`viewMode` 설정.  
- `app/(tabs)/schedule.tsx`: 스크롤 동작 보강, “캘린더에 등록하기” 후 이동 시 전달 파라미터(또는 공유 상태)로 “스케줄 주” 반영.  
- 타임테이블 표시 주: “이번 주” 기본 또는 이동 시 해당 주로 맞추기.  
- 드래그 드롭: 기존 롱프레스+탭 유지, 시각 피드백 강화.

**Out of scope**  
- 진짜 제스처 드래그(pan) 구현(별도 기능으로 추후).  
- 다른 탭(Notes, Inbox) 레이아웃 변경.
