# 인박스 간소화 + 구조화 → 캘린더 — Plan

**Date**: 2026-02-26  
**Project**: Mendly (LifeBalanceAI)

---

## Overview

1. **인박스 간소화**: 쓸데없는 박스 제거, 음성 탭 제거, 라벨 정리로 사용 경로를 명확히 한다.
2. **구조화한 항목 → 캘린더**: 항목 리스트에서 "구조화하기" 후에도 "캘린더에 추가"가 가능하도록 하여, 앱 본질인 플래너(캘린더)로 연결한다.

---

## 목표

- **간소화**: 음성(🎤) 제거, "고정 일정 / 우선 순위 자동 배치" 카드 제거 또는 한 줄로 축소. 구조화하기 = 노트로, Auto-Schedule = 캘린더에 추가임을 라벨로 명시. 빈 상태 CTA는 "일정 만들기" 유지하되 Schedule로 이동하는 것이 명확하거나 "캘린더에서 추가" 보조 CTA 추가.
- **구조화 → 캘린더**: 구조화된 각 항목에 "캘린더에 추가" 버튼 추가. 클릭 시 해당 항목 텍스트로 parse-schedule 호출 → 1개 이상 일정이면 auto-schedule → events insert. parse-schedule은 **최소 1개** 일정까지 허용하도록 완화.

---

## 범위

### In scope

- parse-schedule API 및 lib: 최소 3개 → **최소 1개** 일정 허용.
- Inbox 항목 카드: 구조화(parsed)된 항목에 "캘린더에 추가" 버튼, 로딩/성공/실패 처리.
- Inbox UI: 음성 행 제거, "고정 일정…" 카드 제거, 라벨·빈 상태 정리.

### Out of scope

- 활동–이벤트 매칭 개선, 별도 API 추가.

---

## 성공 기준

- 구조화된 항목에서 "캘린더에 추가"로 1개 이상 일정이 캘린더(events)에 저장된다.
- 인박스 화면에 음성 버튼과 "고정 일정 / 우선 순위 자동 배치" 카드가 없고, 구조화/Auto-Schedule 역할이 라벨로 구분된다.
