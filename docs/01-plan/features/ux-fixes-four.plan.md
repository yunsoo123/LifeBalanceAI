## 1. AI 일정 대화 — “내일” 등 추론 (요일 재질문 방지)

### 목표
- 사용자가 “낼 현수랑 약속”, “저녁 7시쯤”이라고 해도 **요일을 다시 묻지 않고** 금요일 19:00으로 해석.

### 전제
- `lib/ai/schedule-generator.ts`의 `scheduleChat()`에 **오늘 날짜**가 넘어가지 않음.
- 시스템 프롬프트에 “내일/모레/요일 추론” 규칙이 없음.

### 작업
1. **API 호출 시 오늘 날짜 주입**  
   - `app/api/schedule/chat+api.ts`(또는 scheduleChat 호출부)에서 서버의 “오늘” `YYYY-MM-DD`와 요일(한글) 문자열 생성.  
   - `scheduleChat(messages, { today: '2026-02-26', todayLabel: '목요일' })` 형태로 확장하거나, 메시지 목록 앞에 컨텍스트 메시지로 추가.
2. **시스템 프롬프트 보강** (`lib/ai/schedule-generator.ts`)  
   - “오늘은 {date} ({요일})입니다.” 문장 추가.  
   - “사용자가 ‘내일’, ‘모레’, ‘금요일’ 등으로 말하면 이 날짜를 기준으로 해석하세요. **요일을 다시 묻지 마세요.**”
3. (선택) 클라이언트에서 “내일” → “2월 27일 금요일” 등으로 치환한 문장을 보내기.

### 완료 조건
- “낼 현수랑 서울 약속” + “저녁 7시쯤”만으로 요일 재질문 없이 일정 생성.
- `npx tsc --noEmit`, `npx expo lint` 통과.

## 2. 다크모드 텍스트 가독성

### 목표
- 다크 테마에서 **검정/진한색 글씨 제거** → theme 기반 색만 사용.

### 작업
1. **전수 조사**  
   - `text-gray-900`(dark: 없음), `#111`, `#000`, `color: 'black'`, `#0F172A`(텍스트용) 등 검색.
2. **교체**  
   - `theme === 'dark' ? colors.text.primary : colors.text.primaryLight` 또는 기존 design-system 텍스트 토큰으로 통일.
3. **공통 컴포넌트**  
   - Input, EmptyState, MetricCard, TimeWheelPicker 등 이미 `dark:text-gray-100` 등 있는지 확인하고, 빠진 곳 수정.

### 완료 조건
- 다크모드에서 모든 화면·모달에서 텍스트가 읽기 좋은 대비.
- tsc, lint 통과.

## 3. 타임테이블 제목 표시 (에브리타임 스타일)

### 목표
- 캘린더 “이번 주” 타임테이블에서 **일정 블록에 제목이 2줄까지** 보이도록.

### 작업
1. **캘린더 타임테이블 블록** (`app/(tabs)/calendar/index.tsx`)  
   - `numberOfLines={1}` → `numberOfLines={2}`.  
   - 블록 `minHeight` 또는 행 높이를 2줄이 들어갈 만큼 약간 확대(예: 40 → 44).  
   - 폰트 10px 또는 11px로 조정해 2줄이 잘리지 않게.
2. (선택) 긴 제목은 “현수와 서울 약속…”처럼 말줄임 유지.

### 완료 조건
- “현수와 서울 약속”, “노래방” 등이 2줄로 보이거나 말줄임이 자연스러움.
- tsc, lint 통과.

## 4. 리뷰 페이지 — 일정 완료 토글 및 달성률 반영

### 목표
- 리뷰(및 필요 시 캘린더)에서 **해당 주 일정을 “완료”로 표시**하면, **completed_events / achievement_rate**가 즉시 반영되도록.

### 현재
- `completed_events = total_events`로 고정(모든 이벤트를 완료로 간주).
- “완료” 버튼/토글이 없음.

### 작업
1. **스키마**  
   - `events`에 `completed_at TIMESTAMPTZ NULL` 추가(마이그레이션).
2. **주간 통계**  
   - `completed_events` = 해당 주 기간 내 `events` 중 `completed_at IS NOT NULL` 개수.  
   - `review.tsx`의 `loadWeeklyStats()` 등에서 이 조건으로 집계.
3. **UI**  
   - **리뷰**: 해당 주 일정 목록(또는 goalVsActual 하위 일정)에 “완료” 토글/버튼. 탭 시 `events.update({ completed_at: now() })` 또는 `null` 토글 후 통계 재조회.  
   - (선택) **캘린더** 일정 상세/리스트에도 “완료” 표시.
4. **타입**  
   - `database.types.ts` 재생성, Event 타입에 `completed_at` 반영.

### 완료 조건
- 리뷰에서 일정 하나 “완료” 처리 시 해당 주 completed_events 및 achievement_rate(또는 이벤트 완료율)가 올라감.
- tsc, lint 통과.
