# UI/UX 전 페이지 퀄리티 일관화 — Plan (기획)

## 0. 핵심 원칙 (사용자 확인)

- **색상만 맞추는 것이 목표가 아니다.** 색상은 보조 수단일 뿐.
- **현재 디자인이 “메모장 같다”** — 평면적이고 구조·계층·인터랙션이 약해 “앱”처럼 느껴지지 않음.
- **목표**: **카톡처럼, 노션처럼, 틱틱처럼** 만드는 것. 즉, **제품 수준의 구조·인터랙션·계층**을 갖추어 “앱다운” 느낌을 주는 것이 중요함.

---

## 1. 요청 해석

| 표면 | 실제 의도 |
|------|-----------|
| "컬러만 맞추는 게 아니야, 색상이 중요한 게 아니야" | **색 맞추기는 부수적.** 우선순위는 색이 아니라 **구조·인터랙션·계층**. |
| "지금은 그냥 메모장 같잖아" | 단순 텍스트 나열, 플랫한 레이아웃 → **블록·카드·대화·태스크 구조**가 없음. |
| "카톡처럼. 노션처럼. 틱틱처럼 만드는 게 중요" | **참조 제품의 UX 패턴**을 기준으로 함. 카톡=대화·말풍선·입력바, 노션=블록·계층·카드·DB 느낌, 틱틱=태스크·캘린더·리스트·필터. |
| "확인했어?" | 위 이해가 맞다는 확인 요청. → **확인함.** Plan/Design을 이에 맞춰 수정함. |

---

## 2. 참조 제품별로 기대하는 것 (색상 말고 구조·인터랙션)

| 제품 | 기대하는 UX | Mendly에 적용할 포인트 |
|------|-------------|------------------------|
| **카톡** | 대화 스레드, 말풍선 구분, 하단 입력바 고정, 읽음/상태, 단순한 네비 | Schedule: 채팅처럼 대화 흐름·말풍선·고정 입력창. Inbox: 빠른 입력+바로 반영되는 액션. |
| **노션** | 블록 단위 편집, 제목/본문/체크리스트/DB 뷰, 사이드/계층, 드래그·토글 | Notes: 블록형(제목·본문·체크·태그). Review: 카드·메트릭·목표 vs 실제 같은 “뷰” 구조. 전반적으로 **블록·카드 단위**로 정보 묶기. |
| **틱틱** | 태스크 리스트, 캘린더/타임라인 뷰, 필터·태그·due date, 진행·완료 감각 | Calendar: 일정 블록·드래그·타임라인. Inbox→일정: “할 일”이 “일정 블록”으로 보이는 구조. 목표 vs 실제·진행률 등 **태스크/일정 감각**. |

---

## 3. 참조 이미지 (시각 참고 — 우선순위는 구조)

- 이미지 4장은 **시각 참고**로 유지하되, Design에서 **1순위**는 아래와 같이 둠.
- **1순위**: 화면별 **구조·블록·인터랙션** (카톡/노션/틱틱 패턴).
- **2순위**: 카드·타이포·여백·색상 등 시각 정리.

---

## 4. 목표 (수정)

- **“메모장 같다”를 벗어나** 카톡·노션·틱틱처럼 **구조·인터랙션·계층**이 분명한 **앱다운** 디자인으로 전환.
- 색상은 **통일만** 하면 되고, **핵심은 블록/카드/대화/태스크 구조**와 **일관된 인터랙션 패턴**.

---

## 5. 인수 조건

- [ ] Plan/Design에 **“Product feel: 카톡·노션·틱틱”**가 **1순위**로 반영됨(구조·인터랙션·블록 단위).
- [ ] 색상·타이포는 **2순위**로 Design에 위치함.
- [ ] 적용 순서: 구조·패턴 먼저 → 그다음 시각(색·여백) 정리.

- **Design**: “카톡·노션·틱틱” **구조·인터랙션** 스펙을 앞에 두고, 색/카드/타이포는 그 다음에 정의.
