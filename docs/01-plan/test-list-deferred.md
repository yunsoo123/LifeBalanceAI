# 테스트 목록 (배포 직전 작성 예정)

**작성 방침**: Phase 0부터 마지막 Phase까지의 **테스트 목록은 배포 직전**에 일괄 작성한다.  
당분간은 기능/배포 준비를 진행하고, 출시 직전에 이 문서에 구체적인 테스트 케이스를 채운다.

---

## 범위

| Phase | 내용 | 테스트 목록 작성 시점 |
|-------|------|----------------------|
| Phase 0 | Foundation (설정, 스키마, 문서) | 배포 직전 |
| Phase 4 | MVP 검증 | 배포 직전 |
| Phase 5 | 통합 / E2E 플로우 | 배포 직전 |
| Phase 6 | 품질(성능, 접근성, 보안, 테스트 인프라) | 배포 직전 |
| Phase 7 | 한국 결제·무료 한도·인앱 결제 | 배포 직전 |
| Phase 8 | 런치 준비(브랜딩, 법적, EAS/스토어) | 배포 직전 |

---

## 작성 시 참고

- **단위/통합**: `npm test`, `npm run test:coverage` 기준으로 누락 구간 정리
- **E2E**: Detox/Maestro 등으로 Inbox → Schedule → Calendar → Notes → Review + 로그인/구독 플로우
- **인앱 결제**: 개발 빌드에서 Pro 구매·복원·한도 우회 (실기기/샌드박스)
- **수동 체크리스트**: 설치, 로그인, 각 탭 핵심 액션, 다크모드, 오프라인 배너 등
