# 리뷰·노트 탭 현황 및 개선 방안

**Date**: 2026-02-22  
**Project**: Mendly (LifeBalanceAI)

---

## 1. 해석 (의도)

- **표면**: "리뷰탭과 노트탭 현재 개발현황 파악, 기능·디자인 개선 및 사용자 능률 향상 방안 모색"
- **의도**: (1) 두 탭의 **as-is**를 정리하고, (2) **기능·디자인·사용성** 관점에서 구체적 개선안을 도출해, (3) 우선순위를 두고 단계적으로 적용할 수 있게 한다.

---

## 2. 전제

- 리뷰·노트 탭은 이미 실기기 피드백·Phase A/B 반영으로 상당 부분 구현된 상태.
- design 문서(review-tab-ui-ux, notes-ux-and-visual-overhaul, notion-style-notes 등)가 있으나, 일부 체크리스트는 미적용.
- 개선은 **기능 확장**, **시각/일관성**, **업무 효율(플로우·인지 부하)** 세 축으로 나누어 정리.

---

## 3. 리뷰 탭 — 현재 개발 현황

### 3.1 구현된 기능

| 영역 | 내용 |
|------|------|
| **데이터** | 해당 주 `schedules`(planned_hours, activities_json) + `events`(실제 시간·완료 수) + `notes` 개수 + **해당 주 todos**(due_date/created_at 기준) 집계. |
| **요약 카드** | 총 몰입 시간, 계획 달성률(%), 전주 대비 증감, Over Budget 한 줄, **달성률 게이지바**(ProgressBar, height 14, 캡션 completionRate). |
| **목표 vs 실제** | GoalVsActualBar per 활동, 완료한 일정 N/M, 노트 개수. |
| **이번 주 일정** | 해당 주 이벤트 리스트, 완료 토글(클릭 시 completed_at 갱신). |
| **이번 주 할 일** | 해당 주 todos N건, 평균 성취도(achievement_percent) %. |
| **AI 인사이트** | 다음 주 제안 말풍선, 수락/나중에, 인사이트 생성 버튼, 생성된 인사이트 목록(접기 가능). |
| **액션** | ⋮ 메뉴(공유, 복사, 내보내기, 인사이트 생성), 주 선택(◀ 이번 주 ▶). |
| **빈 상태** | 로그아웃 시 로그인 유도, 데이터 없음 시 Schedule로 가기 CTA. |

### 3.2 디자인·일관성

- **reviewCardStyle**로 카드 통일, LAYOUT.sectionTitle 사용.
- 다크 모드, design-system 색·테두리 적용됨.

### 3.3 미적용/미완 (design 대비)

- **진척률 정의 툴팁**: "계획 달성률" 옆 (i) 터치 시 정의 문구(planAchievementRateTooltip) — design에 있으나 미노출.
- **이벤트 완료율 %**: "완료한 일정 N / M" 옆에 "(이벤트 완료율 XX%)" — 선택 항목으로 design에 명시됨.
- **AI 블록**: 이미 하나의 "AI 인사이트" 카드로 통합된 상태. 내부 순서·접기만 추가 정리 가능.
- **Over Budget**: 현재 카드 내 한 줄로 표시됨. "N개 목표 초과" + 탭 시 상세는 미구현(선택).

## 4. 노트 탭 — 현재 개발 현황

### 4.1 구현된 기능

| 영역 | 내용 |
|------|------|
| **목록** | 노트 카드(제목, 내용 요약, 태그, 연결 일정 뱃지, 수정일), 검색, "+ 새 노트" / "첫 노트 만들기" CTA. |
| **상세/편집** | 제목, 본문(문단/체크/코드블록 파싱), 태그, **일정 연결** 블록(항상 노출, design §1-1 반영), AI 요약(Enhance) 카드. |
| **데이터** | notes 테이블(title, content, tags, event_id), loadNotes/createNewNote/saveNote/deleteNote, toggleEventLink. |
| **AI** | POST /api/note/enhance → summary/actionItems/improvements, 토스트·i18n 적용. |

### 4.2 디자인·일관성

- EmptyState, 카드 스타일(rounded-[20px] 등), design-system 적용 구간 있음.
- "일정 연결" 캡션+설명+CTA 항상 노출(design 반영).

### 4.3 미적용/미완

- **저장/삭제 성공**: design에서 Alert 제거 → showSuccess 토스트로 전환 권장. 현재 일부만 토스트일 수 있음.
- **목록→상세 전환**: "한 번의 탭으로 새 노트 열기" 등 플로우는 구현됨. 추가로 제목 포커스 등 세부 UX 가능.
- **지식 연결·AI 요약** 블록이 캡처/캘린더와 완전히 동일한 카드 언어인지 일부 구간 점검 여지 있음.

## 5. 기능적 개선 방안

### 5.1 리뷰 탭

| # | 개선 | 설명 | 우선순위 |
|---|------|------|----------|
| R1 | **진척률 툴팁** | "계획 달성률" 옆 (i) → planAchievementRateTooltip 모달/인라인. | 높음 |
| R2 | **이벤트 완료율 %** | "완료한 일정 N / M" 옆 "(이벤트 완료율 XX%)" (total_events > 0). | 중 |
| R3 | **목표–이벤트 매칭 완화** | 활동명과 이벤트 제목이 다를 때도 키워드/별칭 매핑으로 goalVsActual 채우기. | 중(별도 플랜) |
| R4 | **다음 주 제안 수락 시 반영** | "수락" 시 다음 주 schedule 초안 생성 또는 캘린더에 placeholder 이벤트. | 낮음 |
| R5 | **리뷰 기간 사용자 지정** | 주 단위만이 아니라 "최근 7일" 등 선택 옵션. | 낮음 |

### 5.2 노트 탭

| # | 개선 | 설명 | 우선순위 |
|---|------|------|----------|
| N1 | **저장/삭제 피드백 통일** | 모든 저장·삭제 성공 시 Alert 제거, showSuccess 토스트로 통일. | 높음 |
| N2 | **빠른 필터** | 목록 상단 "전체 / 태그별 / 일정 연결됨" 칩으로 필터. | 중 |
| N3 | **노트→리뷰 링크** | 노트 상세에서 "리뷰에서 보기" → 해당 주 리뷰로 이동(연결된 일정 기준). | 중 |
| N4 | **오프라인 초안** | 저장 실패 시 로컬 초안 보관, 온라인 시 재시도. | 낮음 |
| N5 | **템플릿/스니펫** | 자주 쓰는 노트 형식(회의록, 일일 회고 등) 템플릿 선택 후 본문 초기화. | 낮음 |

## 6. 디자인적 개선 방안

### 6.1 공통

| # | 개선 | 설명 |
|---|------|------|
| D1 | **터치 타겟 44dp** | 모든 버튼·칩·토글 최소 44×44dp 검증 및 부족 구간 수정. |
| D2 | **다크 모드 일관** | 리뷰·노트 전반에서 텍스트/배경이 design-system과 일치하는지 스캔. |
| D3 | **섹션 간격** | 카드 간 mb-5/mb-6 등 일관, 리뷰 탭 스크롤 시 시각적 리듬 유지. |

### 6.2 리뷰

| # | 개선 | 설명 |
|---|------|------|
| D4 | **요약 숫자 강조** | 총 몰입 시간·달성률 %를 더 큰 폰트/색으로 강조해 한눈에 인지 가능하게. |
| D5 | **Over Budget 요약** | 여러 개일 때 "N개 목표 초과" + 탭 시 상세 리스트(모달 또는 접이형). |
| D6 | **주 선택 UI** | ◀/▶ 외에 "이번 주" 한글 표기 등 가독성 개선. |

### 6.3 노트

| # | 개선 | 설명 |
|---|------|------|
| D7 | **목록 카드 정보량** | 제목·요약·태그·연결 뱃지·수정일이 한 카드에서 스캔하기 쉽게 정리. |
| D8 | **편집 화면 구분** | 편집 모드일 때 입력 필드/버튼이 뷰 모드와 명확히 구분되도록 테두리·배경. |
| D9 | **AI 요약 카드** | 캡처/캘린더 카드와 동일한 카드 스타일(radius, padding, border) 적용 여부 확인. |

## 7. 사용자 능률 향상 방안

| # | 개선 | 탭 | 효과 |
|---|------|-----|------|
| U1 | **진척률 정의 바로 확인** | 리뷰 | "이게 뭔가?" 인지 부하 감소 → 툴팁(R1). |
| U2 | **한 화면에서 주요 지표** | 리뷰 | 요약 카드 한 장에 시간+달성률+증감+Over Budget 한 줄 유지, 스크롤 최소화. |
| U3 | **할 일·일정 완료 액션** | 리뷰 | 이번 주 일정/할 일에서 완료 토글·성취도 설정이 리뷰 화면에서 바로 가능(이미 구현). 유지·강조. |
| U4 | **노트 빠른 생성** | 노트 | 첫 노트/새 노트 한 번에 열기(이미 구현). 제목 포커스·키보드 자동 올리기로 입력 지연 감소. |
| U5 | **일정 연결 발견성** | 노트 | "일정 연결" 블록 항상 노출(이미 반영). 설명 문구로 가치 전달 유지. |
| U6 | **리뷰↔노트 이동** | 노트·리뷰 | 노트에서 "리뷰에서 보기" 또는 리뷰에서 "이번 주 노트 N개" 탭 시 노트 목록(필터)으로 이동. |
| U7 | **단축 플로우** | 공통 | 인박스→캡처→할 일/일정 저장 후 "리뷰에서 확인" 토스트 + 링크(선택). |

## 8. 권장 적용 순서

### Phase 1 (Quick — 설계와 일치·일관성)

1. **리뷰**: 진척률 툴팁(R1) — i18n 키 추가 후 (i) 터치 시 툴팁 노출. ✅ (기존 구현)
2. **노트**: 저장/삭제 성공 시 토스트 통일(N1). ✅ (기존 구현)
3. **공통**: 터치 타겟 44dp 점검(D1). ✅ 리뷰 (i) 버튼 min 44dp, ⋮ 메뉴/이벤트 행/Button·Badge 점검 완료; 노트 연결 일정 행에 minHeight 48 적용.

### Phase 2 (Minor — 가독성·효율)

4. **리뷰**: 이벤트 완료율 % 표시(R2). ✅ (기존 구현 확인)
5. **리뷰**: 요약 숫자 강조(D4). ✅ (text-2xl 적용)
6. **노트**: 목록 카드 정보 구조 점검(D7). ✅ (마지막 수정 라벨·메타 구분)

### Phase 3 (기능 확장)

7. **노트**: 목록 필터(태그/일정 연결됨)(N2). ✅ (칩: 전체/일정 연결됨/태그)
8. **노트↔리뷰**: "리뷰에서 보기" / "이번 주 노트" 이동(N3, U6). ✅ (노트→리뷰 버튼, 리뷰 Notes Created 터치→노트)
9. **리뷰**: 목표–이벤트 매칭 완화(R3, 별도 plan/design).

**요약**: 리뷰 탭은 요약·목표 vs 실제·이번 주 일정/할 일·AI 인사이트까지 구현되어 있으며, 진척률 툴팁·이벤트 완료율 % 등 design 미적용 항목이 남아 있음. 노트 탭은 목록·편집·일정 연결·AI 요약이 구현되어 있고, 저장/삭제 피드백 통일·필터·리뷰와의 이동이 개선 포인트. 기능·디자인·능률 개선을 Phase 1~3로 나누어 순차 적용하는 것을 권장함.
